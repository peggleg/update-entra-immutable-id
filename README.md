# Entra ID Identity Remediation Script (Google Workspace IdP Sync)

An administrative PowerShell script leveraging the **Microsoft Graph SDK** to safely update a user's `UserPrincipalName` (UPN) and `OnPremisesImmutableId` (Source Anchor) in Microsoft Entra ID.

---

## рџ› пёЏ The Problem This Script Solves

When managing user lifecycle provisioning where **Google Workspace acts as the Identity Provider (IdP)** and syncs to Entra ID, changing a user's primary email address breaks the identity mapping.

- **The Cause:** The provisioning engine uses the primary email string (or a hash derived from it) as the **Source Anchor** (`OnPremisesImmutableId`) to bind the Google identity to the Entra ID object.
- **The Conflict:** While Google can successfully push updates to the user's Email and UPN fields during a name/email change, Entra ID strictly **blocks direct modifications** to the `OnPremisesImmutableId` while a user is actively bound to a verified production domain namespace.
- **The Result:** Provisioning halts, synchronisation errors are flagged, and Single Sign-On (SSO) fails for that user.

This script automates the standard Microsoft-approved remediation workflow to manually update the immutable ID and restore identity health.

---

## вљ пёЏ Important Warnings

> **CHANGING THE IMMUTABLE ID CAN BREAK SINGLE SIGN-ON FOR THE USER.**
> Ensure you fully understand the implications before proceeding. The script will prompt for explicit confirmation (`YES`) before making any changes.

- Run this script during a low-impact window if possible, as the user's UPN will temporarily change to an `.onmicrosoft.com` address during remediation.

---

## Process Workflow

To bypass Entra ID's attribute locking behaviour, this script executes a **multi-step remediation pattern**:

1. **Step 1 вЂ” Isolate Account:** Changes the user's UPN to a temporary cloud-native domain anchor (`username@your-tenant.onmicrosoft.com`). This moves the user outside the synchronised/federated namespace constraint, which unlocks the `OnPremisesImmutableId` attribute.

2. **Step 2 (Part 1) вЂ” Update Source Anchor:** Directly sets the `OnPremisesImmutableId` to the new value generated by Google Workspace. The script verifies the change against Entra ID before proceeding.

3. **Step 2 (Part 2) вЂ” Re-link & Finalise:** Updates the UPN to the final desired production email/domain string, allowing Google Workspace's provisioning tool to successfully resume delta sync updates.

Between each step the script waits **10 seconds** to allow for Entra ID propagation, then re-fetches and verifies the user object before continuing.

## Prerequisites

### 1. Configuration вЂ” Set Your Intermediate Domain

Before running the script, open `update-immutable-id.ps1` and update the `$IntermediateDomain` variable at the top of the file to match your tenant's `.onmicrosoft.com` domain:

```powershell
$IntermediateDomain = "your-tenant.onmicrosoft.com"
```

You can find your `.onmicrosoft.com` domain in the [Microsoft Entra admin centre](https://entra.microsoft.com) under **Overview > Primary domain**.

### 2. PowerShell Module Installation

The Microsoft Graph PowerShell SDK must be installed on your system:

```powershell
Install-Module Microsoft.Graph -Scope CurrentUser
```

To update an existing installation:

```powershell
Update-Module Microsoft.Graph
```

### 3. Required Microsoft Graph Permissions

The script requests the following delegated scopes at runtime via device code authentication:

| Scope | Purpose |
|---|---|
| `User.Read.All` | Read user profile and current attribute values |
| `User.ReadWrite.All` | Update UPN and `OnPremisesImmutableId` |

The authenticating account must hold a role in Entra ID that grants these permissions (e.g. **User Administrator** or **Global Administrator**).

---

## Usage

### Running the Script

```powershell
.\update-immutable-id.ps1
```

The script uses **device code authentication**. When prompted, open the displayed URL in a browser and enter the code to authenticate.

### Interactive Inputs

After authenticating, the script will prompt for the following three values:

| Prompt | Description | Example |
|---|---|---|
| **Current UserPrincipalName** | The user's existing UPN or Entra ObjectId | `oldname@yourdomain.com` |
| **New On-Premises Immutable ID** | The new Source Anchor value from Google Workspace | `newname@yourdomain.com` |
| **Final UserPrincipalName** | The desired UPN after remediation is complete | `newname@yourdomain.com` |

The script will display a summary of the planned changes and require you to type `YES` to confirm before any modifications are made.

---

## Post-Run Verification

After the script completes, verify the following in the [Microsoft Entra admin centre](https://entra.microsoft.com):

1. Navigate to **Identity > Users** and locate the user.
2. Confirm the **User principal name** matches the final UPN you specified.
3. Under **Properties**, confirm the **On-premises immutable ID** reflects the new value.
